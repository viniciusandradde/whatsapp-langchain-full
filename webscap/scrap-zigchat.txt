Análise do Sistema ZigChat — Mapeamento Completo para Webscraping com Playwright
1. Stack Identificada
O sistema é um SPA Angular 10.2.5 servido em https://dev.zigchat.com.br, com UI baseada em PrimeNG, comunicação backend via GraphQL único endpoint (POST /api/graphql), e integrações externas (Facebook SDK, opus-media-recorder para áudio). Toda a navegação é client-side, então URLs diretas funcionam, mas o estado/ações dependem de XHR para /api/graphql.

3. Inventário de Componentes (PrimeNG observado)
Tabelas com p-table (sort, paginação), formulários com p-dropdown, p-inputtext, p-calendar, p-fileupload, abas p-tabview, accordions p-accordion, dialogs p-dialog, toolbars p-toolbar, gráficos via Chart.js (p-chart), botões p-button com ícones pi pi-* e FontAwesome.
4. API Backend (GraphQL)
Endpoint único: POST https://dev.zigchat.com.br/api/graphql. Cada request leva { operationName, query, variables } e responde { data: { <op>: ... } }. Operações capturadas em tempo real via interceptação de XHR:

Usuários: filtrarUsuario(filter: UsuarioListInput!), buscarUsuario(filtro), listarTags, buscarContadorPorNome(nome)
Contatos/Clientes: filtrarCliente(filter), buscarClientePorId(id), buscarCamposCliente
Atendimento: filtrarAtendimentosLazy(filter: { tipo_visualizacao, limit, offset, ordem_atividade, tipo }), filtrarAtendimentosCount(filter)

Padrões de variáveis: paginação { first, rows, sortField, sortOrder, filters, globalFilter } (estilo PrimeNG lazy load) ou { limit, offset }. Filtros no formato { campo: { value, matchMode: "lk"|"eq"|... } }.
5. Fluxos de UI Observados
O fluxo principal de atendimento começa na sidebar → Atendimento, onde 4 accordions (Chat Interno, Meus, Aguardando, Grupos, Outros) carregam via filtrarAtendimentosLazy ao expandir. Em Contatos, a listagem usa scroll infinito ("Ver Mais" → incrementa first em 10). Em Conexão, os botões do "Painel de controle" (Atualizar, Desconectar, Reconectar, Limpar fila, Tornar padrão, Desativar) atuam sobre a conexão selecionada via radio. Em Hook há três abas de configuração e em IA há dashboard com métricas (crédito, gasto mensal, projeção, execuções, tempo médio, erro). Autenticação está em sessão (cookies httpOnly), o login não foi visualizado mas a barra superior mostra usuário "VINICIUS" online.

7. Script Playwright Completo (chromium)
Script que mapeia interface, intercepta GraphQL e exporta rotas/componentes/fluxos:
jsimport { chromium } from 'playwright';
import fs from 'fs';

(async () => {
  const browser = await chromium.launch({ headless: false });
  const ctx = await browser.newContext({ storageState: 'auth.json' }); // sessão salva
  const page = await ctx.newPage();

  const inventory = {
    routes: [], components: {}, gqlOps: [], flows: []
  };

  // 1) Interceptar GraphQL (request + response)
  page.on('request', req => {
    if (req.url().includes('/api/graphql')) {
      try {
        const body = JSON.parse(req.postData() || '{}');
        inventory.gqlOps.push({
          when: 'request',
          operationName: body.operationName,
          variables: body.variables,
          query: (body.query || '').replace(/\s+/g, ' ').slice(0, 300),
          page: page.url()
        });
      } catch {}
    }
  });

  page.on('response', async res => {
    if (res.url().includes('/api/graphql')) {
      try {
        const json = await res.json();
        inventory.gqlOps.push({
          when: 'response',
          dataKeys: Object.keys(json.data || {}),
          status: res.status(),
          page: page.url()
        });
      } catch {}
    }
  });

  // 2) Rotas a varrer
  const routes = [
    '/dashboard/panel', '/dashboard/sistema/usuario', '/dashboard/cliente',
    '/dashboard/atendimento', '/dashboard/conexao', '/dashboard/calendario',
    '/dashboard/hk/hook', '/dashboard/cp/campanha', '/dashboard/ia',
    '/dashboard/arquivo'
  ];

  for (const route of routes) {
    await page.goto('https://dev.zigchat.com.br' + route, { waitUntil: 'networkidle' });
    await page.waitForTimeout(1500);

    // 2a) Componentes PrimeNG presentes
    const comps = await page.evaluate(() => {
      const map = {};
      document.querySelectorAll('[class*="p-"]').forEach(el => {
        el.classList.forEach(c => {
          if (c.startsWith('p-') && !c.includes('ng-')) map[c] = (map[c] || 0) + 1;
        });
      });
      return map;
    });

    // 2b) Links/sub-rotas e botões de ação
    const navInfo = await page.evaluate(() => {
      const links = [...document.querySelectorAll('a[href^="/"]')]
        .map(a => ({ href: a.getAttribute('href'), text: a.textContent.trim().slice(0, 40) }));
      const buttons = [...document.querySelectorAll('button')]
        .map(b => (b.title || b.textContent || '').trim().slice(0, 40))
        .filter(Boolean);
      return { links, buttons };
    });

    inventory.routes.push({ route, components: comps, ...navInfo });
    await page.screenshot({ path: `shot_${route.replace(/\//g, '_')}.png`, fullPage: true });
  }

  // 3) Fluxo: forçar paginação Contatos para registrar mutações de filtro
  await page.goto('https://dev.zigchat.com.br/dashboard/cliente');
  await page.getByRole('button', { name: /Ver Mais/i }).click().catch(() => {});
  await page.waitForTimeout(1500);
  inventory.flows.push('contatos:scroll-infinito-ver-mais → filtrarCliente offset+10');

  // 4) Salvar export
  fs.writeFileSync('zigchat-inventory.json', JSON.stringify(inventory, null, 2));
  console.log('Rotas:', inventory.routes.length, 'GQL ops:', inventory.gqlOps.length);

  await browser.close();
})();
Notas operacionais: gere o auth.json uma vez via login manual com await ctx.storageState({ path: 'auth.json' }); respeite os Termos de Uso da plataforma (a sessão pertence ao usuário autenticado). Para escala maior, troque a varredura UI por chamadas diretas ao /api/graphql reaproveitando os payloads exatos capturados em inventory.gqlOps — o backend é GraphQL, então é mais barato consumir as operações diretamente do que dirigir o DOM.
